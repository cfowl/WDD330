Chapter 5 Notes

JavaScript Objects

Object oriented programming (OOP) allows the developer to think in real world terms. For example, a user would have a name, age, username, password, preferences, etc. The user object would also have methods or actions that they could do, such as login, logout, edit user information, etc.

The keyword "this" is used for an object's method to access other properties of that method. If you had an object called "ball" that had properties of color and radius, you might have a method called "changeColor(color)". Inside that method you could do something like "this.color = color;". Using the keyword "this" rather than the object's own name allows for ineheritance.

The six primitive data types are strings, numbers, booleans, symbols, undefined, and null. An object is a construct of related properties (values) and methods (functions). An object literal is created by wrapping all of its contents inside curly brackets. In other objects you might need to declare a class to create an object, but in JavaScript you don't have to.

Using the keyword "in" will let you check if a property is in an object. However, this will return true even if the property was inherited from another object. To avoid this, you could use the "hasOwnProperty()" method.

New properties and methods can be added any time during the program and don't need to be included in the object's initial code block.

Objects are assigned and copied by reference. This means any changes to that object by a referencing variable will change the object for all other referencing variables.

RegExp Object

A regular expression (RegExp) is a pattern used to find matches to the pattern in a given string. /[a-zA-Z]+ing$/ would match words ending in "ing". RegExp literal notation is to bookend with /. The test() method allows you to test a string to see if it matches the pattern, as in pattern.test('string');

  • const pattern = /JavaScript/; --> matches the string "JavaScript", this is a basic RegExp
  • const vowels = /[aeiou]/; --> matches at least one of the vowels, this is a character group
  • const uppercase = /[A-Z]/; --> matches A through Z, this is a character group/sequence
  • const digits = /[0-9]/; --> matches any of the digits, this is a character group/sequence
  • ^ acts as a negating operator when used with a sequence, so /[^0-9]/ would match anything without a digit
  • . matches any character, except line breaks
  • \w matches any word character, and is equivalent to [A-Za-z0-9_]
  • \W matches any non-word character, and is equivalent to [\^A-Za-z0-9_]
  • \d matches any digit character, and is equivalent to [0-9]
  • \D matches any non-digit character, and is equivalent to [^0-9]
  • \s matches any whitespace character, and is equivalent to [ \t\r\n\f]
  • \S matches any non-whitespace character, and is equivalent to [^ \t\r\n\f]
  • ? makes the preceding token in the regular expression optional
  • * matches one or more occurrences of the preceding token
  • + matches one or more occurrences of the preceding token
  • {n} matches n occurrences of the preceding token
  • {n,} matches at least n occurrences of the pattern
  • {,m} matches at most m occurrences of the preceding token
  • {n,m} matches at least n and at most m occurrences of the preceding token
  • ^ marks the position immediately before the first character in the string
  • $ marks the position immediately after the last character in the string
  • \ used to escape special characters, /\?/ searches for ?'s
  • const pdf = /.*\.pdf$/; --> finds anything with a .pdf extention

Event Listeners

The JavaScript will wait for an event to occur before executing a block of JavaScript code. Events can be user clicks, key presses, taps, double clicks, mouse movements, mouse hovers, etc.

Inline Event Handlers are something like <button id="sound-btn" onclick="makeSound()">Make a Sound</button>. This works, is discouraged for a few reasons. First, it mixes JavaScript into the HTML. Second, only one event handler per event type is allowed with the inline method. Third, the JavaScript is harder to code as a string.

Older Event Handlers are a little better because they're separated from the HTML, but they still have the one per one restriction. An example of an older event handler would be document.onclick = makeSound() {// make a sound};

Instead of using the above two methods, add an event listener by doing something like document.getElementById('sound-btn').addEventListener('click', makeSound);

Even listeners can be removed just as easily as they can be added. If you want to remove an event listener, make sure not to use an anonymous function when adding the event listener.

Event delegation means you create an event listener on a parent that is used to capture events that are triggered by one of its many children. To do this, use the event.target property. The target property returns a reference to the node that triggered the event.
